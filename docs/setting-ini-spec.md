# `setting.ini` 仕様

`setting.ini` は `csend.exe` と同じフォルダに置く設定ファイルです。起動時に読み込み、終了時に一部の値を保存します。

## 使うセクション

現在使うセクションは次のとおりです。

- `[font]`
- `[Window]`
- `[category]`
- `[notification]`
- `[message]`

## `[font]`

アプリの表示フォントを指定します。

| キー | 型 | 既定値 | 内容 |
| --- | --- | --- | --- |
| `name` | 文字列 | `MS UI Gothic` | リストやコンボボックスに使うフォント名 |
| `size` | 数値 | `16` | フォントサイズ |

### 読み込み時の動作

- `name` が空の場合は既定値 `MS UI Gothic` を使います。
- `size` が空の場合は既定値 `16` を使います。
- `size` が 0 以下、または `name` が空の場合は、既定のフォントを使います。

### 保存時の動作

- 終了時に現在のフォント設定を `[font]` に保存します。

## `[Window]`

ウィンドウの位置とサイズを保存します。

| キー | 型 | 既定値 | 内容 |
| --- | --- | --- | --- |
| `window` | 16進数文字列 | `0000000001000100` | `top`, `left`, `bottom`, `right` を 4 桁ずつ連結した値 |

### 形式

`window` は次の形式です。

```text
TTTTLLLLBBBBRRRR
```

- `TTTT` = 上
- `LLLL` = 左
- `BBBB` = 下
- `RRRR` = 右

### 読み込み時の動作

- 起動時に `window` を読み込み、その値で `MoveWindow` します。
- 値が不正な場合は、既定のウィンドウ位置を使います。

### 保存時の動作

- 終了時に現在のウィンドウ位置から値を作成して保存します。
- 最大化中は、最大化前の通常サイズを保存します。

## `[category]`

カテゴリ一覧と、前回選択したカテゴリ名を保存します。

| キー | 型 | 既定値 | 内容 |
| --- | --- | --- | --- |
| `count` | 数値 | `0` | カテゴリ数 |
| `itemNNN` | 文字列 | なし | カテゴリ名 |
| `pathNNN` | 文字列 | なし | カテゴリのデータ参照先 |
| `last` | 文字列 | 空 | 前回選択したカテゴリ名 |

### `itemNNN` / `pathNNN`

- `NNN` は `000` から始まる 3 桁の連番です。
- `count` の数だけ、`item000` / `path000` から順に読み込みます。
- `itemNNN` はカテゴリ表示名です。
- `pathNNN` はカテゴリが参照するデータソースです。
- `pathNNN` には次のいずれかを指定できます。
- ローカルファイルのパス
- `http://` または `https://` で始まる URL

### `pathNNN` の扱い

- ローカルファイルの場合、アプリは `csend.exe` のあるフォルダを基準に探します。
- 見つからない場合は、指定された文字列をそのままパスとして扱います。
- `http://` または `https://` で始まる場合は URL として扱います。
- URL のカテゴリは参照専用です。

### `last`

- 終了時に、現在選択中のカテゴリ名を `last` に保存します。
- 起動時は `last` と一致するカテゴリ名を選択します。
- 一致するカテゴリがない場合は、先頭のカテゴリを選択します。

## `[notification]`

クリップボード転送時などの通知方法を指定します。

| キー | 型 | 既定値 | 内容 |
| --- | --- | --- | --- |
| `toast` | `on` / `off` | `off` | Windows の toast 通知を使うかどうか |
| `tip_ms` | 数値 | `3000` | カーソル付近に出すアプリ内 tip の表示時間（ミリ秒） |

### `toast`

- `on` のときだけ Windows の toast 通知を表示します。
- `off` のときは toast を表示しません。
- `toast` は OS の通知センターに残る通知です。

### `tip_ms`

- アプリ内の吹き出し通知の表示時間を指定します。
- 既定値は 3000 ミリ秒です。
- `0` 以下の場合は既定値を使います。

### 通知の考え方

- すぐ見せたい反応は、アプリ内の tip 吹き出しで表示します。
- toast は必要な場合だけ補助的に表示します。
- ステータスバーは使いません。

## `[message]`

固定文言を保存します。

このセクションは、アプリ内で表示する短い固定文言の管理に使います。

| キー | 型 | 既定値 | 内容 |
| --- | --- | --- | --- |
| `data_read_failed` | 文字列 | `データファイルを読み込めませんでした。` | データファイルの読み込みに失敗したときの文言 |
| `data_too_large` | 文字列 | `データファイルが大きすぎます。上限は %d バイトです。` | データファイルがサイズ上限を超えたときの文言 |
| `data_binary_ng` | 文字列 | `バイナリ形式のデータファイルは使えません。` | バイナリ扱いで拒否したときの文言 |
| `data_format_ng` | 文字列 | `データファイルの形式が正しくありません。` | 3行1レコード形式が崩れているときの文言 |
| `data_count_ng` | 文字列 | `データファイルは100件までです。` | 件数上限を超えたときの文言 |
| `data_write_failed` | 文字列 | `データファイルを書き込めませんでした: %s` | データファイルの書き込みに失敗したときの文言 |
| `data_write_error` | 文字列 | `データファイルの書き込み中にエラーが発生しました: %s` | 保存中の例外やエラー時の文言 |
| `data_add_limit` | 文字列 | `データは100件までです。` | 追加操作で件数上限に達したときの文言 |
| `url_read_failed` | 文字列 | `URLから読み込めませんでした。` | URL 取得失敗時の文言 |
| `read_failed` | 文字列 | `読み込みできませんでした。` | 汎用の読み込み失敗文言 |

### 読み込み時の動作

- キーが存在する場合は、その値をそのまま使います。
- キーが存在しない場合は、既定の日本語文言を使います。
- キーが存在しない場合のみ、起動時にそのキーを `setting.ini` に自動追記します。
- 既存のキー値は上書きしません。
- そのため、古い `setting.ini` を使っているユーザーでも、新しい文言キーだけ自動で補完されます。

## 設定例

```ini
[category]
count=2
item000=Git
path000=samples\en\Git.txt
item001=SQL
path001=https://raw.githubusercontent.com/NaruSue/c-send/feature/codex_1.6/samples/jp/SQL.txt
last=Git

[font]
name=MS UI Gothic
size=16

[Window]
window=04EE0982080C0C4F

[notification]
toast=on
tip_ms=3000

[message]
data_read_failed=データファイルを読み込めませんでした。
data_too_large=データファイルが大きすぎます。上限は %d バイトです。
data_binary_ng=バイナリ形式のデータファイルは使えません。
data_format_ng=データファイルの形式が正しくありません。
data_count_ng=データファイルは100件までです。
data_write_failed=データファイルを書き込めませんでした: %s
data_write_error=データファイルの書き込み中にエラーが発生しました: %s
data_add_limit=データは100件までです。
url_read_failed=URLから読み込めませんでした。
read_failed=読み込みできませんでした。
```

## 補足

- `pathNNN` の中身はカテゴリ設定画面で編集できます。
- データ保存形式を将来増やす場合でも、`setting.ini` は参照先の管理に集中させます。
- `type` のような項目は、現時点では `setting.ini` ではなくデータファイル側で扱います。

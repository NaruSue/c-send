# `setting.ini` から参照されるデータファイル仕様

この文書は、`setting.ini` の `[category]` セクションで `pathNNN` に指定されるデータファイルの仕様をまとめたものです。

## 対象

- `setting.ini` の `[category]` セクション
- `path000`, `path001` などで指定されるファイル
- 1つのカテゴリに対応する定型文データ

## ファイルの役割

各カテゴリは 1 つのデータファイルを持ちます。

- `itemNNN` はカテゴリ名
- `pathNNN` はそのカテゴリに対応するデータファイルのパス

アプリはカテゴリ選択時に `pathNNN` を読み込み、データ一覧として表示します。

## パスの解決

アプリはまず `csend.exe` があるフォルダを基準に `pathNNN` を探します。

1. `csend.exe` のあるフォルダ + `\` + `pathNNN`
2. 1 で見つからなければ、`pathNNN` をそのまま開く

つまり、`pathNNN` には次のどちらも指定できます。

- 相対パス
- 絶対パス

未存在のファイルを選んだ場合でも、保存時には新規作成できます。

## ファイル形式

データファイルは 1 件につき 3 行で構成されます。

```text
name
type
value
```

### 1 件の構成

- 1 行目: 表示名
- 2 行目: 種別番号
- 3 行目: 本文

これを繰り返して複数件を保存します。

## 各行の仕様

### 1 行目: `name`

- 一覧に表示される名前です
- 読み込み時に前後の空白は `Trim()` されます
- 空行は無視されます

### 2 行目: `type`

- 整数値です
- データの中身の種類を表します
- 現時点では `0` = text を使います
- 将来は `1` = jpeg のように、別形式を追加できます

### 3 行目: `value`

- クリップボードに送る本文です
- 読み込み時に逆エスケープされます
- 前後の空白は保持されます

## エスケープ仕様

`value` は 1 行テキストとして保存するため、次の文字がエスケープされます。

- `\` -> `\\`
- 改行 -> `\n`
- 復帰 -> `\r`
- `,` -> `\,`

読み込み時は上記の逆変換を行います。

## 読み込み時の挙動

- ファイルが存在しない場合は、空の一覧として扱います
- 途中で 3 行そろっていない場合でも、読めた範囲で項目化します
- `name` が空の項目は追加されません

## 書き込み時の挙動

- ファイルは上書き保存されます
- 全項目を先頭から 3 行ずつ書き出します
- 保存時にファイルがなければ新規作成されます

## 記述例

```text
Git Status
0
git status

Git Pull
0
git pull
```

## `setting.ini` との対応例

```ini
[category]
count=2
item000=Git
path000=samples\en\Git.txt
item001=JSON
path001=C:\Users\example\Documents\csend\custom-json.txt
```

## 補足

- `pathNNN` の内容はカテゴリ編集画面で変更できます
- データファイルはカテゴリごとに別管理です
- クリップボード送信用の定型文を保存する用途を想定しています
- `type` はデータ本体の形式を表し、`pathNNN` の種類とは別です
